# 豆包（字节跳动）— 图像生成参考

> 平台：https://www.doubao.com/chat/
> 模型：Seedream 4.5（默认文生图模型）
> 页面状态：需要在聊天界面中点击"图像生成"按钮切换到图片模式
> 测试日期：2026-06-15

## 关键差异（对比 Gemini/ChatGPT）

| 项目 | 豆包 | Gemini | ChatGPT |
|:----|:----|:-------|:--------|
| 生图模式 | 独立"图像生成"子界面，不是聊天内嵌 | 聊天内嵌 | 聊天内嵌 |
| 每次出图数 | **3~4 张** | 1 张 | 1 张 |
| 下载方式 | 批量下载（推荐）：「...」→「下载」→界面蓝色「下载」按钮 | "Download full size image"按钮 | "Save"按钮 |
| 模型选择 | 有下拉菜单可选模型 | 聊天顶部切换 | 聊天顶部切换 |
| 可选参数 | 比例（1:1/16:9/9:16/4:3/3:4/3:2/2:3）| 无 | 无 |
| 可选参数 | 风格（写实/动漫/插画/像素风等）| 无 | 无 |
| 下载文件名 | 自动根据标题命名为 `{标题}.png` | `Gemini_Generated_Image_{hash}.png` | `ChatGPT Image {日期}.png` |
| 原图分辨率 | **1773×2364**（远超其他平台） | 1024×559 | 1024×1024 |

## 工作流

### Step 1: 进入图像生成模式

1. 在豆包聊天界面底部工具栏找到"图像生成"按钮并点击
2. 界面切换为图像生成模式：顶部显示 **"图像生成"** 标题
3. 左上角可选的模型通常是 **Seedream 4.5**（支持下拉切换）
4. 可选的参数：
   - **比例**：1:1（默认）、16:9、9:16、4:3、3:4、3:2、2:3
   - **风格**：写实、动漫、插画、像素风等
5. 输入框占位文本变为 **"描述你想要的图片"**

### Step 2: 输入提示词

直接输入提示词并回车即可。提示词中英文都支持。

### Step 3: 等待生成

- 生成中显示 **"正在为您生成图片..."**
- 出图时间约 **30~90 秒**
- 生成完成后显示 **3~4 张图片**（缩略图，带"正在为您生成图片..."文字仍可能残留）

### Step 4: 下载原图

豆包有两种下载方式，**推荐用方式A（批量下载）**：

#### ✅ 方式 A：批量下载（推荐，一次下完所有图）

1. 在图片下方的操作栏中找到 **「...」更多按钮**（expandable haspopup="menu"），点击
2. 弹出菜单中选择 **"下载"**
3. 页面打开 **「批量下载」** 界面，默认所有图片已勾选
4. 底部显示 **「已选择 N 项内容」**（N 为生成张数）
5. 底部右侧找到蓝色 **「下载」** 按钮，点击
6. Chrome 自动开始下载，每张图保存为独立文件
7. 文件命名格式：`{对话标题}.png`、`{对话标题} (1).png`、`{对话标题} (2).png`……
8. 下载后的文件在 `C:\\Users\\Administrator\\Downloads\\` 目录下

> 💡 批量下载的要点：**两步点击**——先「更多」→「下载」打开批量界面，再点界面中的蓝色「下载」按钮。缺一步都不行。

#### 备选：方式 B — 预览灯箱单张下载

> ❗ 豆包的下载流程跟 Gemini/ChatGPT 不一样，不要找按钮文本"Download full size"或"Save"

1. **点击其中一张图片**（用 `evaluate_script` 的 `img.click()` 或 snapshot 的 uid click）
2. 页面进入**预览灯箱模式**：右侧大图预览，右侧边缘有缩略图导航
3. 右上角找到**蓝色下载按钮**（图标为白色向下箭头）
4. 点击下载按钮
5. 文件下载到 `C:\\Users\\Administrator\\Downloads\\{标题}.png`

> ⚠️ 方式 B 只能单张下载，如果要一次下完所有图建议用方式 A。

### Step 5: 通过 NapCat 发送

```python
import requests
src = r"C:\Users\Administrator\Downloads\拥挤地铁惊艳女子.png"
file_uri = "file:///" + src.replace("\\", "/")
payload = {"user_id": 2415317075, "message": [{"type": "image", "data": {"file": file_uri}}]}
r = requests.post("http://127.0.0.1:18801/send_private_msg", json=payload, timeout=30)
```

## 注意事项

- 点击图片打开预览后，右下角可能没有明显的"下载"文字，但右上角的蓝色下载按钮是标准操作
- 下载的文件名取决于对话标题（左侧历史对话列表中的标题）
- **批量下载的文件命名规则**：第一张 `{对话标题}.png`，后续 `{对话标题} (1).png`、`{对话标题} (2).png`……
- 如果第一次点击下载没反应或文件没出现到 Downloads，检查网络请求 tab 看是否有 download 触发了，或再次点击按钮
- 豆包的三张图是不同角度/构图，不是完全相同——这跟 GPT-Image2 的 4 宫格预览不同，豆包每次出多张实质不同的变体
- 2026-06-15 实测，豆包出图数量会在 3~4 张之间浮动（两次测试：地铁主题3张，仙侠主题4张）

## DOM 快照识别

进入图像生成模式后，snapshot 中可识别的元素：
- 输入框：uid 含有 textbox 属性，值为 `multiline`，占位文本 "描述你想要的图片"
- 模型选择按钮：文本为 "Seedream 4.5" 的 button
- 比例选择按钮：文本为 "比例" 的 button
- 风格选择按钮：文本为 "风格" 的 button
- 预览灯箱下载按钮：snapshot 中 complementary 区域里的 button，通常是该区域最后一个可点击元素

## 原始 CDN 地址处理

豆包页面的缩略图 URL 带处理参数：
```
https://{cdn}.byteimg.com/tos-cn-i-a9rns2rl98/rc_gen_image/{hash}.jpeg~tplv-a9rns2rl98-downsize_watermark_1_5_b.png?params
```

`~tplv-a9rns2rl98-downsize_watermark_1_5_b.png` 是缩略图+水印处理模板。直接去掉该后缀的 URL 无法访问原图。**必须通过批量下载或预览灯箱的下载按钮获取原图**，不要尝试从 CDN URL 拼凑原图。
